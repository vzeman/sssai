### Phase 2: Security Assessment
- Vulnerability scanning: `nuclei -u {target} -severity critical,high,medium -j -o /output/nuclei.json`

- SSL deep scan: `sslyze {target} --json_out /output/sslyze.json`

- Security headers: `drheader scan single {target} --json > /output/drheader.json`
- CORS testing: `python3 corsy.py -u {target} -j > /output/corsy.json`

### Phase 3: Web Application Testing
- OWASP ZAP: `zap-cli quick-scan -s xss,sqli {target}`
- Black-box fuzzing: `wapiti -u {target} -o /output/wapiti/ -f json`

- SQL injection: `sqlmap -u '{target}' --forms --batch --crawl=2 --technique=BEU --risk=1 --output-dir=/output/sqlmap/`

### Phase 4: Secret & Supply Chain
- Secret scanning: `trufflehog filesystem . --json > /output/trufflehog.json`

- SAST analysis: `semgrep scan --config p/security-audit --json --output /output/semgrep.json .`

### Phase 5: Subdomain Takeover & Phishing

### Phase 6: Performance & Load Testing

- Mobile Lighthouse: `lighthouse {target} --emulated-form-factor mobile --output json --output-path /output/lighthouse-mobile.json --chrome-flags="--headless --no-sandbox"`

### Phase 7: SEO & Accessibility
